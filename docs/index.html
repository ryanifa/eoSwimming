How it works

Your swim data starts at t = 0 (the moment the eo paddles started recording). Your video starts at its own t = 0. The offset is how many seconds into the video the swim data should begin: if the first stroke happens at 5.20 s into the video, set swim_start_seconds = 5.20.

Once you've found the value, run the Composite video with overlay workflow in Actions and paste this number. The server will render the same overlay you see here and burn it into the video.
